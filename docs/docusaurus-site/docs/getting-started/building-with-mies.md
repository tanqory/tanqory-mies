# Building with Tanqory Mies

At Tanqory we say, **"Mies is the floor, not the ceiling."**  
The design system gives you strong building blocks, and it's up to you to create experiences that fit your users' needs.  
Don't limit ideas to the existing components too soon. First, design the best solution—then see whether Mies already provides what you need.  
If there's a gap, contribute back to make the system better.

## Zoom Out
Before jumping into solutions, step back and understand the full product context.  
A problem rarely exists in isolation, so look at how it connects to the entire Tanqory platform.

For example, if you're improving **Orders**, study how merchants move through the **Tanqory admin** as a whole.  
Merchants don't care about internal teams—they just use the platform end-to-end, so we must design with the entire experience in mind.

## Explore Freely
Once you understand the big picture, explore without constraints.  
Start with a blank page instead of jumping straight to UI kits or existing components.  
Sketch multiple approaches and pick the one that truly solves the problem, not just the one that matches today's patterns.

**Design principles to guide exploration:**
- **Clarity over cleverness** – Make the path forward obvious
- **Minimal cognitive load** – Reduce mental effort required
- **Consistent with platform** – Feel native to Tanqory ecosystem
- **Accessible by default** – Work for all merchants regardless of ability

## Calibrate with Mies
After refining ideas, align with Mies components and tokens.  
This isn't about constraining creativity—it's about leveraging proven solutions and maintaining platform consistency.

**Check these elements:**
- **Colors**: Use only brand (`#E1FF00`), black (`#000000`), and white (`#FFFFFF`)
- **Spacing**: Follow 4px increments (4, 8, 12, 16, 24, 32px)
- **Typography**: System fonts with standard sizes (14, 16, 18, 24, 32px)
- **Components**: Enhance existing components or create new ones that follow Mies patterns

**When to extend the system:**
- Missing component that solves a common merchant need
- Existing component doesn't handle your specific use case
- New pattern could benefit other teams across Tanqory

## Build and Iterate
Implement your solution using Mies building blocks.  
Start simple, test with real merchants, then refine based on feedback.

**Development approach:**
1. **Build minimum viable version** using existing Mies components
2. **Test with real merchant scenarios** – not just edge cases
3. **Measure success** – task completion, error rates, satisfaction
4. **Iterate based on data** – let merchant behavior guide improvements

## Contribute Back
When you enhance or extend Mies, share improvements so everyone benefits.  
This keeps the system evolving while ensuring unified merchant experiences.

**How to contribute:**
- **Internal teams**: Discuss proposals in the **#mies** Slack channel
- **External developers**: Open a **GitHub discussion** or **issue** in the [Tanqory Mies repository](https://github.com/tanqory/tanqory-mies)
- **Share learnings**: Document what worked, what didn't, and why

**What makes a good contribution:**
- Solves a real merchant problem (not just developer convenience)
- Follows Mies design principles and constraints
- Includes documentation and usage examples
- Has been tested with real users

## Remember the Goal
Great merchant experiences happen when we balance **system consistency** with **user needs**.  
Mies provides the foundation—your job is to build something merchants actually want to use.
