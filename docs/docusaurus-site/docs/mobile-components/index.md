# Mobile Components

Native, cross-platform components optimized for React Native mobile applications with minimal design and intuitive touch interactions.

## Component Philosophy

Our mobile components follow Tanqory Mies principles adapted for mobile:
- **Touch-first design** - Optimized for finger interactions and gestures
- **Cross-platform consistency** - Works seamlessly on iOS and Android
- **Performance-focused** - Efficient rendering and smooth animations
- **Accessible by default** - Mobile accessibility features built-in
- **Minimal visual design** - Clean aesthetics that adapt to platform conventions
- **Developer-friendly** - React Native patterns with clear APIs

## Component Categories

### **[Actions](actions/button)**
Touch-interactive elements that trigger operations
- Button, Floating action button, Swipe actions

### **[Layout & Structure](layout/safe-area-container)**
Containers and organizational elements for mobile screens
- Safe area container, Card, Grid, Stack, Scroll container, Bottom sheet

### **[Inputs](inputs/text-field)**
Mobile-optimized form controls and data entry

### **[Navigation](navigation/app-bar)**

### **[Feedback & Status](feedback/snackbar)**
Mobile-specific status communication and user feedback
- Snackbar, Toast, Banner, Progress indicator, Activity spinner, Pull to refresh

### **[Media & Display](media/avatar)**
Visual content and media handling for mobile

### **[Lists & Data](lists/flat-list)**
Optimized data display for mobile devices

### **[Utilities](utilities/theme-switcher)**
Mobile-specific system utilities and configuration
- Theme switcher, Modal, Popover tooltip, Gesture handler examples

## Mobile Usage Principles

1. **Design for touch** - Ensure adequate touch targets (minimum 44pt)
2. **Consider platform patterns** - Follow iOS and Android conventions
3. **Test on devices** - Verify performance and usability on real devices
4. **Support accessibility** - Include voice over and screen reader support

Each component includes mobile-specific usage guidelines, accessibility notes, platform considerations, and React Native implementation examples to help you build better mobile experiences efficiently.
