# Web Components

Modern, accessible components optimized for web applications with minimal design and clear functionality.

## Component Philosophy

Our web components follow Tanqory Mies principles:
- **Essential functionality** - Each component solves specific user needs
- **Minimal visual design** - Clean aesthetics that don't distract
- **Accessible by default** - WCAG compliance built-in
- **Responsive behavior** - Works across all screen sizes
- **Developer-friendly** - Clear APIs and predictable behavior

## Component Categories

### **[Actions](actions/button)**
Interactive elements that trigger operations
- Account link, Button, Button group, Split button

### **[Layout & Structure](layout/box)**
Containers and organizational elements
- Bleed, Block stack, Box, Callout card, Card, Divider, Empty state, Form layout, Grid, Inline grid, Inline stack, Layout container, Media card, Page shell, Sticky header

### **[Selection & Input](input/text-field)**
Form controls and data entry components
- Autocomplete, Checkbox, Choice list, Color picker, Combo box, Date picker, Drop zone, Filters, Form, Inline error, Radio group, Range slider, Select, Tag, Text field, File uploader, Rich text editor

### **[Images & Icons](media/icon)**
Visual content and symbolic elements
- Avatar, Icon, Keyboard key, Thumbnail, Video thumbnail, Image gallery

### **[Feedback Indicators](feedback/badge)**
Status communication and user feedback
- Badge, Banner, Exception list, Progress bar, Skeleton, Spinner, Toast, Notification center

### **[Typography](typography/text)**
Text rendering and formatting components
- Text, Headings, Code block

### **[Tables](tables/data-table)**
Structured data display and interaction
- Data table, Index table, Virtualized table

### **[Lists](lists/list)**
Collection display and organization
- Action list, Description list, List, Option list, Resource list

### **[Navigation](navigation/link)**
Movement and wayfinding components
- Footer help, Link, Pagination, Tabs, Sidebar, Top bar

### **[Overlays](overlays/modal)**
Contextual and modal interfaces
- Popover, Tooltip, Modal, Drawer

### **[Utilities](utilities/app-provider)**
System-level and configuration components
- App provider, Collapsible, Scrollable, Theme switcher

## Usage Principles

1. **Choose the right component** - Match component to user need and context
2. **Follow accessibility guidelines** - Use semantic HTML and ARIA labels
3. **Test across devices** - Ensure responsive behavior works
4. **Maintain consistency** - Use same components for similar functions

Each component includes usage guidelines, accessibility notes, and implementation examples to help you build better user experiences efficiently.
