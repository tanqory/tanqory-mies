# Common Actions

Standard interaction patterns for frequently used operations that create predictable user experiences.

## Primary Actions

### Create and Add
**Starting new items or relationships**

- **"Create new"** - Starting fresh content (documents, projects, accounts)
- **"Add item"** - Including existing items in collections
- **"Upload file"** - Bringing external content into the system
- **Consistent placement** - Primary position in interface hierarchy
- **Clear outcomes** - Users understand what gets created

### Save and Persistence
**Preserving user work and changes**

- **Auto-save behavior** - Background preservation without user action
- **Explicit save** - User-controlled persistence with clear feedback
- **Save states** - "Saving...", "Saved", "Save failed" with recovery options
- **Draft handling** - Preserve incomplete work across sessions

### Delete and Remove
**Eliminating content or relationships**

- **Destructive confirmation** - "Are you sure?" with consequences explained
- **Undo capabilities** - Recent deletions recoverable
- **Batch operations** - Delete multiple items efficiently
- **Soft delete** - Move to trash before permanent removal when possible

## Secondary Actions

### Edit and Modify
**Changing existing content**

- **Inline editing** - Direct manipulation in context
- **Modal editing** - Dedicated editing interface
- **Bulk editing** - Modify multiple items simultaneously
- **Version awareness** - Handle concurrent edits gracefully

### Copy and Duplicate
**Reusing existing content**

- **Deep vs shallow copy** - Clear about what gets duplicated
- **Naming conventions** - "Copy of [name]" or similar clear patterns
- **Reference handling** - Maintain or break relationships as appropriate

### Share and Export
**Making content available to others**

- **Permission levels** - View, edit, admin access clearly differentiated
- **Link sharing** - Public URLs with appropriate access controls
- **Export formats** - Multiple options when relevant (PDF, CSV, etc.)
- **Recipient management** - Add, remove, modify access for individuals

## Navigation Actions

### Search and Filter
**Finding content in collections**

- **Global search** - Across entire system
- **Scoped search** - Within specific contexts or collections
- **Filter combinations** - Multiple criteria applied simultaneously
- **Search suggestions** - Help users refine queries
- **Clear filters** - Easy return to unfiltered state

### Sort and Organize
**Arranging content for better browsing**

- **Multiple sort options** - Date, name, relevance, custom
- **Sort direction** - Ascending/descending with clear indicators
- **Drag and drop** - Manual reordering when appropriate
- **Categorization** - Tags, folders, or other grouping mechanisms

## Contextual Actions

### Quick Actions
**Frequently needed operations in context**

- **Favorite/bookmark** - Mark items for easy return
- **Copy link** - Share direct access to specific items
- **Print** - Output to physical or PDF format
- **Refresh** - Update data without full page reload

### Batch Operations
**Acting on multiple items simultaneously**

- **Select all** - Quick selection of entire collections
- **Select patterns** - By type, date, or other criteria
- **Bulk actions** - Delete, move, tag, or modify multiple items
- **Progress feedback** - Show completion for long-running operations

## Responsive Action Patterns

### Mobile Adaptations
- **Swipe actions** - Delete, archive, favorite via gestures
- **Long press** - Context menus and additional options
- **Touch targets** - Larger buttons and clear spacing
- **Simplified menus** - Prioritize most important actions

### Desktop Enhancements
- **Keyboard shortcuts** - Power user efficiency
- **Context menus** - Right-click access to actions
- **Hover states** - Progressive disclosure of options
- **Multiple selection** - Shift+click and Ctrl+click patterns

## Feedback and Confirmation

### Success States
- **Immediate feedback** - Actions provide instant response
- **Toast notifications** - Temporary confirmations that don't interrupt flow
- **Status updates** - Persistent indicators for ongoing processes
- **Achievement recognition** - Celebrate completed goals when appropriate

### Error Handling
- **Clear error messages** - Specific problems with suggested solutions
- **Retry mechanisms** - Easy ways to attempt failed actions again
- **Fallback options** - Alternative approaches when primary actions fail
- **Error prevention** - Validation and warnings before problems occur

## Accessibility for Actions

### Keyboard Navigation
- **Tab order** - Logical progression through actionable elements
- **Enter and Space** - Consistent activation patterns
- **Escape key** - Cancel or close action dialogs
- **Arrow keys** - Navigate within action groups

### Screen Reader Support
- **Action labels** - Clear descriptions of what each action does
- **State announcements** - "Saving", "Deleted", "Added to favorites"
- **Result descriptions** - What happened after actions complete
- **Error announcements** - Problems and suggested fixes read aloud

### Visual Indicators
- **Loading states** - Progress indicators for actions taking time
- **Disabled states** - Clear indication when actions aren't available
- **Focus indicators** - Keyboard users can see active elements
- **High contrast** - Actions remain visible in accessibility modes

Consistent action patterns reduce cognitive load and help users confidently accomplish their goals across different parts of your product.
