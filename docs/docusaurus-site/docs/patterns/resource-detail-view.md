# Resource Detail View

Comprehensive display patterns for individual items that balance information density with usability.

## Detail View Structure

### Information Architecture
**Organize content for efficient scanning and comprehension**

- **Primary information** - Most important details prominently displayed
- **Supporting context** - Additional details available without overwhelming
- **Related items** - Connections to associated content
- **Action hierarchy** - Primary and secondary actions appropriately placed

### Layout Patterns
- **Hero section** - Key identifying information and main actions
- **Content sections** - Logical groupings with clear headings
- **Sidebar information** - Metadata, status, or secondary details
- **Footer actions** - Less frequent operations at bottom

## Content Organization

### Progressive Information Disclosure
**Present details at appropriate levels of depth**

- **Summary first** - Essential information visible immediately
- **Expandable sections** - Detailed information on demand
- **Tabbed organization** - Multiple content types (Details, History, Settings)
- **Related content** - Links to associated items or references

### Contextual Information
- **Status indicators** - Current state clearly displayed
- **Timestamps** - Creation, modification, and activity dates
- **Attribution** - Who created, modified, or owns the resource
- **Relationships** - Parent items, dependencies, or associations

### Content Hierarchy
- **Visual emphasis** - Important information stands out
- **Scannable structure** - Headers and spacing aid navigation
- **Consistent patterns** - Similar information types presented uniformly
- **White space usage** - Breathing room prevents cognitive overload

## Interactive Elements

### Primary Actions
**Most common operations prominently available**

- **Edit/Modify** - Change resource content or properties
- **Share/Collaborate** - Provide access to others
- **Export/Download** - Extract resource in various formats
- **Duplicate/Clone** - Create similar resources quickly

### Secondary Actions
- **Archive/Delete** - Remove or hide resources
- **Move/Organize** - Change location or categorization
- **Print** - Physical or PDF output
- **History/Versions** - Access to previous states

### Contextual Menus
- **Overflow actions** - Less common operations
- **Quick actions** - Frequent operations as shortcuts
- **Bulk operations** - Act on related items simultaneously
- **External integrations** - Connect with other tools

## Responsive Behavior

### Mobile Adaptations
**Optimize for smaller screens and touch interaction**

- **Stacked layout** - Vertical arrangement of content sections
- **Collapsible sections** - Hide secondary information initially
- **Touch-friendly actions** - Larger buttons and clear tap targets
- **Simplified navigation** - Fewer simultaneous options visible

### Desktop Advantages
- **Multi-column layouts** - Efficient use of horizontal space
- **Persistent navigation** - Always-visible action bars
- **Hover interactions** - Additional information on mouse over
- **Keyboard shortcuts** - Power user efficiency features

### Tablet Considerations
- **Hybrid approach** - Balance between mobile and desktop patterns
- **Orientation awareness** - Adapt to portrait vs landscape viewing
- **Gesture support** - Swipe navigation between sections
- **Mixed interaction** - Both touch and keyboard/mouse support

## Loading and Performance

### Content Loading Strategies
**Manage performance while providing good user experience**

- **Progressive loading** - Show available content immediately
- **Skeleton screens** - Placeholder content during loading
- **Lazy loading** - Load non-critical sections on demand
- **Caching strategies** - Reduce repeated network requests

### Error Handling
- **Partial failures** - Show available content even if some sections fail
- **Retry mechanisms** - Easy ways to reload failed content
- **Offline behavior** - Cached content when network unavailable
- **Graceful degradation** - Reduced functionality better than broken interface

## Accessibility Guidelines

### Screen Reader Navigation
**Structure content for assistive technology**

- **Heading hierarchy** - Logical H1, H2, H3 structure
- **Landmark roles** - Main content, navigation, complementary sections
- **Skip links** - Jump to important sections quickly
- **Content descriptions** - Alt text for images, labels for form elements

### Keyboard Navigation
- **Tab order** - Logical progression through interactive elements
- **Focus management** - Clear visual indicators for keyboard users
- **Shortcuts** - Quick access to common actions
- **Modal handling** - Focus trapped in dialogs and overlays

### Visual Accessibility
- **Color contrast** - Text readable against backgrounds
- **Focus indicators** - Clear visual feedback for interactive elements
- **Scalable text** - Works with browser zoom and system font sizing
- **Motion sensitivity** - Respect reduced motion preferences

## Related Content Integration

### Cross-References
**Help users discover related information**

- **Similar items** - Resources with comparable characteristics
- **Recent activity** - Recently viewed or modified related content
- **User connections** - Items shared with or created by same people
- **Category browsing** - Other items in same classification

### Contextual Suggestions
- **Next actions** - Logical follow-up operations
- **Related workflows** - Connected processes or tasks
- **Learning resources** - Help documentation or tutorials
- **External references** - Links to relevant external content

### Activity Integration
- **Recent changes** - Timeline of modifications
- **User interactions** - Comments, shares, or collaborations
- **System events** - Automated actions or integrations
- **Analytics insights** - Usage patterns or performance metrics

Well-designed detail views help users understand, interact with, and act upon individual resources efficiently while maintaining awareness of broader context and relationships.
