# Style & Grammar

Consistent style and grammar create professional, trustworthy experiences across all platforms.

## Grammar Essentials

### Sentence Structure
- **Use active voice** - "You can edit this" not "This can be edited"
- **Keep sentences short** - 20 words maximum for clarity
- **Avoid redundancy** - "First priority" not "first priority"

### Punctuation Rules
- **Serial comma** - "Red, white, and blue" (always use the Oxford comma)
- **No periods in UI elements** - Buttons, labels, short phrases don't need periods
- **Consistent quotation marks** - Use straight quotes in code, curly quotes in prose

### Capitalization
- **Sentence case for most UI** - "Add new item" not "Add New Item"
- **Title case for proper nouns** - "Tanqory Mies Design System"
- **Consistent brand names** - Always "GitHub", never "Github"

## Style Guidelines

### Numbers and Dates
- **Spell out one through nine** - "Three items" but "10 items"
- **Use numerals for UI** - "3 of 10 completed"
- **Date format: Month DD, YYYY** - "January 15, 2024"
- **Time format: 12-hour with AM/PM** - "2:30 PM"

### Contractions and Formality
- **Use contractions sparingly** - "Cannot" instead of "can't" in error messages
- **Match the context** - Casual for tips, formal for legal text
- **Consistent tone** - Don't mix formal and casual in the same experience

### Technical Terms
- **Define on first use** - "API (Application Programming Interface)"
- **Use familiar alternatives** - "Sign in" not "authenticate"
- **Consistent terminology** - Pick one term and stick with it

## Writing Patterns

### Headings and Titles
- **Use descriptive headings** - "Payment Methods" not "Options"
- **Parallel structure** - All headings follow same grammatical pattern
- **Hierarchy clarity** - H1 for page titles, H2 for sections, H3 for subsections

### Lists and Bullets
- **Parallel structure** - All items start with same part of speech
- **Logical order** - Chronological, alphabetical, or by importance
- **Consistent punctuation** - Either all sentences or all fragments

### Links and Actions
- **Clear link text** - "View pricing plans" not "click here"
- **Action-oriented buttons** - "Create account" not "account creation"
- **Consistent labeling** - Same action, same words across platform

## Common Style Issues

### Avoid These Patterns
- **Please and thank you overuse** - One per interaction maximum
- **Exclamation point excess** - Save for genuine celebration
- **Jargon without definition** - Always explain technical terms
- **Passive voice** - "Your file was deleted" → "We deleted your file"

### Preferred Constructions
- **Positive framing** - "Complete these steps" not "Don't skip these steps"
- **User-focused** - "You can" not "Users can"
- **Solution-oriented** - Lead with what to do, not what went wrong

## Platform Considerations

### Mobile Adaptations
- **Shorter phrases** - Space is limited
- **Essential information first** - Users scan quickly
- **Finger-friendly actions** - "Tap" not "click" on mobile

### Global Considerations
- **Simple sentence structure** - Easier to translate
- **Cultural neutrality** - Avoid idioms and cultural references
- **Clear context** - Provide enough detail for translation

Consistency in style and grammar builds user confidence and reduces cognitive load across the entire experience.
